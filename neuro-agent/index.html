지배 요인 ①: 코호트 지름길

비-AD 병인은 100% NACC(ADNI는 0). 코호트가 "AD냐 비-AD냐"의 거의 완벽한 지름길이 된다. §3 코호트-중립 통제와 §5 leak probe · shortcut-error가 이걸 방어한다.

지배 요인 ②: 희소 셀

희소 병인 셀이 작다. DLB_LBD · FTLD · VaD · OTHER는 primary 기준 수십 건, test 셀은 한 자릿수(spec §12). 희소 클래스는 bootstrap CI로 보고하고 탐색적 결과임을 명시한다.

산출물 · 라벨

산출물은 두 테이블이다. diffdx_episodes(스캔당 1행)는 학습이 쓸 수 있는 최대 표본이고, diffdx_primary(환자당 1행)는 다중 스캔 환자가 지표를 부풀리지 않게 평가에 쓴다. split은 환자 ID 해시로 배정하므로 스캔 단위로 학습해도 환자 누수가 없다.

라벨 2컬럼:
  step1  cog_status ∈ {CN, CI}
  원 라벨 dx_detail  ∈ {CN, MCI, AD, DLB_LBD, FTLD, VaD, NPH, OTHER}

exp1 step2 = 4-class 접기 (exp1/data.py _STEP2_MAP):
  AD → AD    DLB_LBD → LBD    FTLD → FTLD    VaD·NPH·OTHER → OTHER
  MCI → None  (step2 라벨 없음 → step2 채점에서 빠짐)

step2 타깃 (primary):  AD 1,021 · LBD 47 · FTLD 51 · OTHER 53 (VaD 29 + OTHER 24)
모델 입력 피처 = 임상 7개(age · sex · education_years · MMSE · MoCA · CDR_global · CDR_SoB) + 형태계측 W-score 285개(mri_w/*). W-score는 robust-CN 참조군(NACC 1,451 + ADNI 501, ≥3년 안정 정상)으로 자체 적합한다(외부 온라인 정규화 툴 미사용). CSF · APOE 바이오마커는 선택(ADNI만 존재).
03 Input Rendering

kv → 자연어 vignette

exp1/render.py가 케이스를 자연어로 만든다. kv 벡터가 아니라 임상 vignette로 LLM에 준다.

render_vignette

나이·성별·교육 + 인지선별(MMSE · MoCA · CDR global/SoB) + render_note가 만든 CDR 6박스 · NPI-Q 12증상 · FAQ 10항목 · GDS-15 서술.

"73세 M, 교육 12년. 인지선별: MMSE 24/30, MoCA 19/30,
 CDR 전역 0.5 (SoB 2.0). ..."   ← 뒤에 척도 서술이 붙는다
render_mri

형태계측 W-score를 RadiologistAgent 내러티브로 렌더한다. VOLUME/AREA는 빼고 VA(피질두께 프록시, 신경퇴행 신호가 가장 강한 계열) 95영역을 전량 넣는다. 좌우 영역을 엽별로 묶어 위축 심한 순으로 서술하고, FreeSurfer 라벨코드는 한글 영역명으로 매핑한다(미지 코드는 라벨<code> 폴백). VA가 없으면 명명 5영역 z로 폴백하고, MRI 자체가 없으면 (구조 MRI 미가용). → 실제 렌더 출력은 §6 참조.

코호트-중립 통제 (렌더 단계)

EXCLUDED_KEYS가 코호트-배타·식별 필드를 LLM 입력에서 뺀다: csf_* · apoe_e4(ADNI만), urinary_incontinence(NACC만), 그리고 cohort · id · days_delta. 없는 항목은 코호트와 무관하게 동일 문구로 렌더한다(문장 미기재). 결측 패턴 자체가 코호트 지문이므로 표현 통일만으로는 부족하고 배타 필드 제거가 핵심이다.

04 Experiment Design

2-step 캐스케이드 + 고정 4단계 프로토콜

step1은 CN/CI 선별, step2는 4-class 병인. 그 위에 고정 4단계 프로토콜을 얹는다 (exp1/pipeline/runner.py StagedRunner.run_case).

STAGE A
선별
CN인지 CI인지 판정. CN이면 여기서 종료(step2 없음).
STAGE B
MRI 전 감별
병인을 하나로 좁히지 않고 순위매긴 감별. 앵커링 완화 스캐폴드이자 측정 지점이고, 채점하지 않는다.
STAGE C
MRI 재순위
구조 MRI W-score 서술을 반영해 감별을 재평가한다.
STAGE D
최종
병인 1개 + 확신도 + 자유텍스트 검사권고.
앵커링 측정

B의 top-1(ranking_b)과 D의 예측(step2_pred)이 얼마나 바뀌는지로 앵커링을 측정한다. B에서 순위를 열어두고 C에서 MRI로 재순위할 여지를 남기는 것이 스캐폴드의 목적이다.

메모리: ExpeL ⊕ ReasoningBank

메모리는 두 축을 합친다(exp1/memory/). ExpeL은 과거 증례를, ReasoningBank는 거기서 추출한 감별 전략을 맡는다. 둘 다 매 단계 프롬프트에 주입되지만, 갱신(learn)은 train에서만 일어난다.

경험 풀 · ExpeL
완료된 케이스를 저장한다. 검색 키는 vignette다(정답·바이오마커 아님). 새 케이스가 오면 vignette 유사도로 과거 증례를 retrieve해 "과거 유사 증례" 블록으로 붙인다. 예: "78세 여성, 반복 환시·인지 변동·RBD, 해마 보존 → 부검확정 LBD".
전략 뱅크 · ReasoningBank
추출한 규칙을 3-필드로 관리한다: 제목 · 사용 조건 · 지침(steering). 항목마다 [success] / [failure] 태그와 vote가 붙어 신뢰도를 표시한다.
전략 아이템 3-필드 + 큐레이션 연산
[success] 환시·변동 동반이면 DLB core 우선
  · 사용 조건 : 조기 환시·인지변동·RBD·파킨소니즘 동반 시 (순수 기억상실형엔 미적용)
  · 지침      : 해마 상대보존을 근거로 AD보다 DLB를 상위에 두라
  · votes=2

큐레이션 연산 (증류 LLM 출력, 케이스당 최대 4개):
  ADD      새 전략 추가
  AGREE    기존 전략에 동의 → votes+1
  EDIT     기존 전략 미세 수정
  REMOVE   잘못된 전략 삭제
gold 게이트 증류 · failure-first-class

에피소드가 끝나면 gold(부검·임상 확진) 정답으로 성공/실패를 판정해 증류한다. 모델이 스스로 맞다고 보는 self-judge가 아니다. 성공이면 전이 가능한 전략을, 실패면 같은 실패를 막을 counterfactual 예방 교훈을 뽑아 [failure]로 남긴다. 실패를 1급 신호로 쓰는 것이 ReasoningBank의 핵심이고, §5 ablation(성공+실패 vs 성공-only)이 이 가치를 직접 검증한다.

주입은 format_memory_block이 전략 뱅크와 과거 유사 증례를 매 단계 프롬프트에 붙이고, 끝에 "각 항목이 이 증례에 적용되는지 먼저 판단하라"는 스티어링을 단다. 학습은 offline train→freeze다: train에서만 learn(뱅크·풀 갱신)하고 eval은 주입만 한다(주입 자체는 train·eval 모두). 학습곡선은 체크포인트마다 eval 지표를 train 에피소드 수에 대해 찍는다.

동결 모델: Qwen3.5-122B (open-weight self-host)

동결 모델은 self-host open-weight를 쓰고 로컬 vLLM/SGLang 엔드포인트에 물린다. 모델선택 벤치는 완료했다. Qwen3.5-122B Top-1 0.282 · macro-F1 0.169 (테스트한 오픈 모델 중 최고).

05 Evaluation · Ablation

무엇을 재고, 무엇을 비교하는가

지표는 exp1/eval/에 있다. 지표·러너 기계는 있으나 ablation 드라이버 스크립트도, 실험 결과도 아직 없다.

지표
분류지표
Primarystep2 balanced macro-F1 학습곡선 (memory-on vs off)
Secondarystep1 balanced accuracy · 클래스별 recall(희소) · 앵커링율(B→D 재순위) · shortcut-error(NACC-AD 슬라이스 오답이 비-AD로 새는 비율) · cohort-probe AUC
희소 클래스percentile bootstrap CI로 보고
cohort-probe는 1회 offline이고, AUC가 0.5에 가까우면 누출 채널이 없다는 근거다.
Ablation (spec §9)
  • memory off vs on (헤드라인).
  • 성공+실패 vs 성공-only. ReasoningBank 본 논문의 49.7 vs 46.5를 우리 도메인에서 재현해 failure-first-class의 가치를 직접 검증한다.
  • (선택) exemplar-only vs strategy-only vs both.
  • (선택) one-shot vs staged.
